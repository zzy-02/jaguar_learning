关于i2c-tools中大多数板级 / BMC / CPLD / RC 模式相关操作，只用下面 5 个命令就够了
```bash
i2cdetect   看总线 / 找设备
i2cdump     看寄存器全貌
i2cget      读单个寄存器
i2cset      写寄存器（配置 RC 模式常用）
i2ctransfer 高级读写（少量场景）
```

### i2cdetect   看总线 / 找设备
- 查看当前系统中有哪些i2c总线`i2cdetect -l`
- 扫描某条总线(4)上的设备`i2cdetect -y 4`

### i2cdump     看寄存器全貌
- 强制(-f)查看(dump)总线4(4)上0X74的寄存器状态：`i2cdump -f -y 4 0x74`

### i2cget      读单个寄存器
- `i2cget -y 4 0x74 0x10`
    >总线 4
    设备 0x74
    寄存器 0x10

### i2cset      写寄存器（配置 RC 模式常用）
`i2cset -y <bus> <addr> <reg> <value>`
>ps:这里的RC value 的“语义来源”只有三个：板卡原理图、CPLD 寄存器说明文档、厂商 bring-up 指南。根据公司产品确定

示例：打开 RC 模式
`i2cset -y 4 0x74 0x10 0x01`
向0x74设备,0x10寄存器，写入0x01(RC enable)
验证：`i2cget -y 4 0x74 0x10`


### 概念解释
- RC：Root Complex，是 PCIe 拓扑中“发起端”的角色，负责枚举、配置和管理 PCIe 设备。
常用的`lspci`本质就是RC枚举的结果
为什么要配置RC模式，因为有些硬件设备(SoC / DPU / 智能网卡)既可以当RC(主机)，也可以当EP(endpoint卡 / 从设备)，所以要明确角色

- CPLD：Complex Programmable Logic Device，CPLD 是板级“可编程控制芯片”，用于管理和控制硬件状态。写在这里主要用于模式选择（RC / EP）
```
CPLD 决定硬件工作模式
 → 硬件以 RC 模式启动
 → OS 才能作为 PCIe 主机运行


RC = PCIe 主控角色
EP = PCIe 从设备
CPLD = 板级“开关控制器”
```
